# Weather CLI

Интерактивное командное приложение для получения информации о погоде на основе Open-Meteo API. Приложение позволяет просматривать текущую погоду, почасовой прогноз и прогноз на несколько дней вперед.

## 📋 Описание проекта

**Weather CLI** — это легковесное, быстрое приложение на Go, которое предоставляет информацию о погоде в выбранном городе через командную строку. Приложение использует бесплатный API Open-Meteo (без ключа API) и поддерживает кэширование данных для оптимизации производительности.

## ✨ Возможности

- 🌍 **Информация о текущей погоде** — температура, влажность, давление, видимость, направление ветра
- 📊 **Почасовой прогноз** — температура и вероятность осадков на следующие часы
- 📅 **Ежедневный прогноз** — минимальная и максимальная температура, вероятность осадков
- 💾 **Кэширование данных** — локальное кэширование с TTL для снижения нагрузки на сервер
- 🔄 **Переключение городов** — возможность динамически менять город в интерактивном меню
- 🔁 **Повторные попытки подключения** — автоматические повторные попытки при сетевых ошибках
- ⚙️ **Конфигурация** — поддержка конфигурационного файла для сохранения города по умолчанию

## ⚙️ Описание конфигурации

### Файл конфигурации

Конфигурация хранится в файле `config.yaml` в корневой директории проекта.

#### Структура `config.yaml`

```yaml
default_city: Protvino
```

- **`default_city`** (строка) — город, который будет загружен при запуске приложения по умолчанию

### Редактирование конфигурации

Вы можете отредактировать файл `config.yaml` вручную или использовать интерактивное меню приложения для изменения города.

Пример:
```yaml
default_city: Moscow
```

## 📖 Руководство по использованию

### Запуск приложения

```bash
./weather-cli
```

### Интерактивное меню

После запуска приложение отображает текущую погоду и интерактивное меню с командами

### Примеры команд

#### Просмотр текущей погоды

```bash
./weather-cli
```

#### Просмотр почасового прогноза

Нажмите `1` для просмотра почасового прогноза на следующие 24 часа.

#### Просмотр ежедневного прогноза

Нажмите `2` для просмотра ежедневного прогноза на 7 дней.

#### Смена города

Нажмите `c` для изменения города:
```
Введите название города:
```

Введите название нового города и нажмите Enter.

#### Обновление данных

Нажмите `r` для обновления информации о погоде (очищается локальный кэш).

#### Выход

Нажмите `q` для завершения работы приложения.

## 🏗️ Обзор архитектуры (опционально)

### Структура проекта

```
weather-cli/
├── main.go                      # Точка входа приложения
├── config.yaml                  # Конфигурационный файл
├── go.mod                       # Зависимости Go
├── go.sum                       # Хэши зависимостей
└── internal/
    ├── app/
    │   └── app.go              # Основная логика приложения и интерактивный цикл
    ├── cache/
    │   └── cache.go            # Система кэширования с TTL
    ├── config/
    │   └── config.go           # Загрузка и сохранение конфигурации
    ├── domain/
    │   └── models.go           # Модели данных (Today, HourlyEntry, DailyEntry)
    ├── provider/
    │   ├── provider.go         # Интерфейс провайдера погоды
    │   └── openmeteo/
    │       ├── openmeteo.go    # Реализация клиента Open-Meteo
    │       └── dto.go          # DTO для десериализации API ответов
    ├── retry/
    │   └── retry.go            # Логика повторных попыток подключения
    └── ui/
        └── ui.go               # Рендеринг пользовательского интерфейса
```

### Ключевые компоненты

- **Application** (`internal/app/app.go`) — главный контроллер приложения, управляет интерактивным циклом
- **TTLCache** (`internal/cache/cache.go`) — кэширование данных с автоматическим истечением
- **Config** (`internal/config/config.go`) — управление конфигурацией приложения
- **WeatherProvider** (`internal/provider/provider.go`) — интерфейс для различных провайдеров данных о погоде
- **Client** (`internal/provider/openmeteo/openmeteo.go`) — реализация провайдера Open-Meteo API
- **UI** (`internal/ui/ui.go`) — компонент отображения интерфейса

### Поток данных

1. Приложение загружает конфигурацию из `config.yaml`
2. Инициализируется Open-Meteo клиент и TTL кэш
3. Приложение отображает текущую погоду для города по умолчанию
4. Пользователь взаимодействует через интерактивное меню
5. Запросы к API кэшируются для оптимизации производительности
6. При сетевых ошибках автоматически выполняются повторные попытки подключения

## ❓ Часто задаваемые вопросы

### Как изменить город по умолчанию?

1. Отредактируйте `config.yaml` и измените значение `default_city`
2. ИЛИ запустите приложение и нажмите `c` в интерактивном меню

### Почему приложение требует интернет-соединение?

Приложение получает данные о погоде из Open-Meteo API, поэтому требует активного интернет-соединения.

### Могу ли я использовать это приложение оффлайн?

Нет, приложение требует интернет-соединение для получения данных о погоде. Однако кэширование помогает минимизировать количество запросов.

### Как долго сохраняется кэш?

TTL кэша определен в `internal/cache/cache.go`. Обычно данные кэшируются на определенный период времени для оптимизации производительности.
